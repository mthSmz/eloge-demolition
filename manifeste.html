Position éditoriale

Manifeste

Un laboratoire critique simple : deux angles, un même texte, zéro dogme.

Pourquoi ce dispositif existe

Éloge / Démolition part d’une idée volontairement sobre : il n’y a que des lectures situées, argumentées, discutables.

Le site met donc en scène deux voix contradictoires pour rappeler que toute œuvre ou sujet peut toujours être défendu et contesté de manière systématique.

La fabrication d'un réel à la demande est désormais automatisé et ouvre la voie à un solipsisme numérique renforcé.

Comment on limite les hallucinations

Patterns utilisés pour limiter les hallucinations et garantir la qualité du contenu généré :

Sortie structurée (Structured Output / JSON Schema)

Contexte : génération intermédiaire contrainte (observations, citations, interprétations) pour rendre la sortie vérifiable avant rédaction finale.

Ancrage par citation (Citation Grounding)

Contexte : critique littéraire sous contrainte. Toute affirmation importante doit pointer vers des extraits exacts du texte source.

Validation d’entités (NER Grounding)

Contexte : anti-hallucination nominale. Les noms propres/entités mentionnés doivent exister dans l’entrée.

Audit déterministe (Deterministic Validator)

Contexte : pipeline de prod. Le script d’audit vérifie structure + exact match, sans jugement flou ni modèle secondaire.

Rejet immédiat (Fail Fast)

Contexte : qualité de sortie. Si une règle échoue, on rejette la génération au lieu de “réparer” silencieusement.

Tests par fixtures (Golden/Fixture Tests)

Contexte : régression. Jeux PASS/FAIL pour s’assurer que l’audit continue de détecter citations invalides et structures cassées.

Hors scope Phase 1 : RAG, NLI, SBERT

Contexte : cadrage produit. Ces patterns sont connus dans la communauté, mais volontairement exclus tant que le noyau minimal n’est pas stabilisé.

Message implicite (et assumé)

Ce projet de portfolio n’essaie pas de “faire plus” : il essaie de faire proprement. Définir un cadre, séparer production et validation, documenter les règles, tester les sorties, puis itérer sans gonfler le scope. C’est une démonstration de méthode autant qu’un objet éditorial.
